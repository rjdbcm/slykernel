Memory_Copy/memmove [ Library ]

NAME

 memmove

FUNCTION

Copies N storage units from area starting at S2 to area starting at S1 without any check for buffer overflow. The difference between this memmove and memcpy is that with memmove, the storage areas may overlap (forwards or backwards) and the result is correct (i.e. it is as if S2 is first moved to a temporary area, and then this area is copied to S1 in a separate step).

INPUTS

 Dest := Address to move data to.
 Src  := Address to move data from.
 N    := Number of bytes to move.
